# Contributing

## Workflow

1. Create or reference a task in `tasks/`
2. Work on a branch
3. Open a pull request
4. Request review
5. Merge only after review is completed

## Branch naming

- `feat/<scope>`
- `fix/<scope>`
- `chore/<scope>`
- `docs/<scope>`

## Pull request rules

Every PR should include:

- objective
- linked task(s)
- summary of changes
- test status
- risks / open questions
- screenshots if UI changes

## Task logging

- Add new work to `tasks/backlog.md`
- Create task detail files for larger items in `tasks/logs/`
- Update status when work starts, changes, or completes

## Code review

- No self-merge without review
- Reviewer checks scope, correctness, tests, and rollback risk
- Follow-up work should be tracked in backlog, not hidden in review comments
