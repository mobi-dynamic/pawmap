# PawMap PRD

## 1. Problem

Dog owners often cannot tell whether a venue or destination is dog-friendly until they arrive, and rules are frequently inconsistent, unclear, or outdated.

## 2. Product vision

PawMap helps users search a place on a map and quickly understand the dog policy, rule details, and confidence level before they go.

## 3. Primary user

- Dog owners planning visits to cafes, restaurants, shops, parks, and destinations

## 4. Core user story

- As a dog owner, when I search for a place, I want to know whether I can bring my dog and what rules apply, so I can plan with confidence.

## 5. MVP scope

- Search a place by keyword or map area
- Place details page
- Dog policy status
- Structured rule fields
- Source and verification timestamp
- User-submitted updates
- Admin moderation workflow
- Favorites / saved places

## 6. Out of scope

- General pet support beyond dogs
- Social feed / community network
- Bookings or reservations
- Full trip itinerary planning
- Native mobile apps in v1

## 7. Success metrics

- Time to answer policy question under 10 seconds
- First 100 verified places in target launch area
- User-submitted corrections accepted within moderation SLA
- Returning users save places and revisit search

## 8. Open questions

- Which launch geography comes first?
- Which places API gives the best cost/quality trade-off?
- What source hierarchy defines trusted verification?
- What moderation SLA is realistic for early stage?
