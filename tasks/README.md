# Task Tracking

This directory is the project execution log.

## Files

- `backlog.md` — active and upcoming work
- `release-checklist.md` — release readiness items
- `logs/` — detailed task records for major workstreams

## Rules

- Every meaningful task must be logged
- Update status when a task starts, changes, blocks, or completes
- Reference task IDs in pull requests and commits when practical
